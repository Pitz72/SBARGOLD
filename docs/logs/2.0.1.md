# 🪵 Log 2.0.1 - Implementazione Lexer

**Data**: 2025-12-05
**Versione**: 2.0.1
**Autore**: Antigravity

## 📝 Descrizione
Implementato il primo componente fondamentale del nuovo core: il **Lexer** (o Tokenizer).
Questo componente è responsabile della trasformazione del codice sorgente grezzo in una sequenza di token strutturati, eliminando la necessità di parsing ripetitivo delle stringhe durante l'esecuzione.

## 🛠️ Modifiche Tecniche
- Creato modulo `sbargold_core/lexer.py`.
- Definita classe `Token` per rappresentare le unità sintattiche.
- Implementata classe `Lexer` che supporta:
    - Comandi SBARGOLD standard (es. `SBARGOLD!`, `SBARGOLD?`).
    - Numeri (interi e float).
    - Stringhe.
    - Identificatori (variabili).
    - Operatori e blocchi.
    - Commenti (ignorati correttamente).

## ✅ Verifica
- Creato script di test `tests/test_lexer.py`.
- Il test conferma che il Lexer riconosce correttamente tutti i tipi di token principali.
